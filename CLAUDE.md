# Claude Code Configuration for kessel-sdk-py

This file configures Claude Code for the kessel-sdk-py repository. For architectural guidance, code patterns, and development conventions, see @AGENTS.md and the domain-specific guideline files it references.

## Development Commands

Before committing changes, Claude Code must run these commands in sequence:

```bash
# Format code (excluding generated protobuf files)
black --exclude '.*_pb2(_grpc)?\.py' src/ examples/

# Lint code (excluding generated protobuf files)
flake8 --exclude '*_pb2.py,*_pb2_grpc.py' src/ examples/

# Run tests
pytest
```

All three checks must pass. These mirror the CI workflows (`.github/workflows/lint.yml` and `.github/workflows/test.yml`).

## Pre-commit Hook

If a pre-commit hook fails:
- Fix the issue it reports
- Re-stage fixed files
- Create a NEW commit (never amend unless explicitly requested)

## File Creation Policy

- **NEVER create `*_pb2.py`, `*_pb2_grpc.py`, or `*.pyi` files.** These are generated by `buf generate` and will be overwritten.
- **ALWAYS prefer editing existing files** over creating new ones.
- When adding new SDK modules, create corresponding test files in `tests/test_<module>.py`.

## Commit Behavior

- Run formatting, linting, and tests BEFORE attempting to commit.
- Never use `--no-verify` to skip the pre-commit hook.
- Never use `git add -A` or `git add .`; stage specific files by name to avoid accidentally committing `.env`, credentials, or build artifacts.

## Installation

When working in this repository for the first time, install development dependencies:

```bash
pip install -e ".[dev,auth]"
```
